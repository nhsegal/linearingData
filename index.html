Determine the Mathematical Relationship

In math class we typically label the horizontal axis of a graph with x and the vertical axis with y. In physics class, we need to modify this convention a little to avoid confusion when we communicate. For example, if we believe the relationship between the elapsed time t that an object falls for and the distance d that it falls are related according to the equation d = ½at2, where a is a constant, and gather a set of data measuring d for various values of t, we might want to examine it by plotting d vs. t. We also might find it helpful to plot d vs. t2. This latter type of plot is sometimes unfamiliar to students. Why is it helpful? What sort of shape would we expect if the data fit our proposed equation? This website is intended to helpful students answer such questions.
The graph initially displays a set of data where the dependent variable is plotted against the independent variable. Two variables are related by some power law. (i.e. They fit the equation y=AxB.) The data also has a little bit of noise in it. Notice how the plot changes when you select different horizontal axes for the same dataset. One plot should be nearly linear, and you can determine the equation of the line of best fit. This equation also yields the power law that describes the relationship between the independent and dependent variables.

After submitting the equation for the line of best fit on the transformed data, you can select [Independent Variable] as the horizontal axis. The equation for line you determined also determines the curve that best fits this graph. Returning to our earlier example, if d is directly proportional to t2 a plot of d vs. t2 is linear and a plot of d vs. t has a parabolic shape. It is a lot easier to judge the linearity of a plot, draw a straight line on it, and determine its slope than it is to judge how closely a plot resembles a parabola, draw a parabola of best fit through the data, and determine that curve's equation. This is one reason we sometimes plot transformations of data.
